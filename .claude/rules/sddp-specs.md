# SDD Pilot - Specification File Conventions

When editing feature-artifact files inside `specs/<feature-folder>/`, follow the canonical preservation, format, and section rules in `.github/skills/artifact-conventions/SKILL.md`.

These rules do NOT apply to project-level docs (`specs/prd.md`, `specs/sad.md`, `specs/dod.md`, `specs/project-plan.md`) - see `.claude/rules/sddp-bootstrap.md`. Standalone ADRs under `specs/adrs/` also receive `.claude/rules/sddp-adrs.md`.
